zig and julia are growing but still niche in terms of Claude Code sessions.

Common CLI tools you'd invoke with ocaml:
ocaml, ocamlopt, dune (build tool, very standard), opam (package manager)
dune in particular is what Claude Code would actually run — it's the dominant build tool and you'd see dune build, dune test, dune exec constantly in an OCaml project.

Haskell — the obvious sibling to OCaml. ghc/ghci, and then cabal and stack as the build/package tools. stack build, stack test is what you'd actually see in sessions, much like dune build.
Elixir/Erlang — mix is the one that matters. mix test, mix deps.get, mix compile. It's the dune/cargo equivalent. elixir, iex, erl exist but mix dominates actual CLI usage.
Dart/Flutter — dart and flutter. Growing fast because of Flutter's mobile/web reach. flutter build, flutter test, dart analyze.
Database CLIs — this whole category is absent from your lists: psql, mysql, sqlite3, redis-cli, mongosh. Claude Code hits these regularly when working on backend projects — running migrations, checking schemas, seeding data.
Version managers — nvm, pyenv, rbenv, rustup. These come up when Claude Code needs to switch or verify runtime versions.
Monorepo tools — nx, turbo, lerna

Bundlers & dev servers — this is the most glaring omission. vite, webpack, esbuild, rollup, parcel. In web projects Claude Code runs vite dev, vite build constantly. These are as common as tsc or jest.
ORM & migration CLIs — prisma (prisma migrate, prisma generate), sequelize-cli, typeorm, alembic, knex. Adjacent to your database CLIs but distinct. Prisma especially is everywhere in Node backend work.
E2E testing — playwright, cypress. Different category from unit test runners. npx playwright test comes up a lot.
Serverless & deploy CLIs — vercel, netlify, fly, wrangler (Cloudflare Workers). Real projects use these and Claude Code does invoke them.
Debugging — gdb, lldb, strace. Niche for Claude Code but not zero.
Protobuf — protoc, buf. Relevant in gRPC-heavy codebases.
Utility commands you skipped — xargs (surprisingly absent given how common find | xargs is), chmod/chown, base64, sha256sum/md5sum, mktemp, realpath, dirname/basename, file. These are the plumbing Claude Code uses constantly but nobody thinks to list.
Git adjacent — pre-commit, git-lfs, husky. Not git itself but invoked alongside it.
Wasm toolchain — wasm-pack, wasm-opt. Niche but growing.
Of these, bundlers and xargs are the most "how did we miss that" ones. Prisma is probably next.
